  • Block Level Elements

    • Always takes up full width available
    • Always starts on a new line
    • Browser automatically adds space(margin) before and after
    Element Value
    <div> Defines a division
    <p> Defines a paragraph
    <address> - Defines the contact information for the author/owner of a document or an article.
    - Can be an email address, URL, physical address, phone number, social media handle, etc
    - Text in the <address> element usually renders in italic, and browsers will always add a line break before and after the <address> element.
    <article> - Independent, self-contained content
    - Should make sense on its own and it should be possible to distribute it independently from the rest of the site.
    - Does not render as anything special in a browser. However, you can use CSS to style the <article> element
    <aside> - Defines some content aside from the content it is placed in
    - Should be indirectly related to the surrounding content
    - Often placed as a sidebar in a document
    - Does not render as anything special in a browser, can use CSS to style it
    <blockquote> - Section that is quoted from another source
    - Browsers usually indent <blockquote> elements
    - Use <q> for inline (short) quotations.
    - Cite: specifies the source of the quotation (URL, book citation, etc.)
    <canvas> - Used to draw graphics, on the fly, via scripting (usually JavaScript)
    - <canvas> tag is transparent, and is only a container for graphics, you must use a script to actually draw the graphics
    - Any text inside the <canvas> element will be displayed in browsers with JavaScript disabled and in browsers that do not support <canvas>
    - Height & Width: Default 150px height, 300px width
    <dl> - Used to define a Description List
    - Used with <dt> to define a description term, and <dd> to define a description definition
    - Also see
    <dt> - Used to define a Description Term/Name
    - Used with <dl> to define a description list, and <dd> to define a description definition
    - Can put paragraphs, line breaks, images, links, lists, etc.
    - Also see
    <dd> - Used to define a Description Definition
    - Used with <dl> to define a description list, and <dt> to define a description term/name
    - Can put paragraphs, line breaks, images, links, lists, etc.
    - Also see
    <fieldset> - Used to group related elements in a form
    - draws a box around the related elements
    - Attributes: disabled - Specifies that a group of related form elements should be disabled
    - Attributes: form - a form-id Specifies which form the fieldset belongs to
    - Attributes: name - a text that specifies a name for the fieldset
    - See also
  • Fieldset & Legends
  • Fieldset & Legends - l
    • <figure> - Used to mark up a photo/image in a document, and can uses a <figcaption> to define a caption for the photo/image
    - Specifies self-contained content, like illustrations, diagrams, photos, code listings, etc
    - <figcaption> element is used to add a caption
    <figcaption> - Defines a caption for a <figure> element.
    - Can be placed as the first or last child of the <figure> element.
    <header> - Container for introductory content or a set of navigational links
    - Typically contains:
    • one or more heading elements (<h1> - <h6>)
    • logo or icon
    • authorship information
    - There can be several <header> elements in one html doc.
    - Cannot be placed within a <footer>, <address> or another <header> element
    <main> - Specifies the main content of a document
    - Ccontent inside the <main> element should be unique to the document
    - There must not be more than one <main> element in a document
    - Must NOT be a descendant of an <article>, <aside>, <footer>, <header>, or <nav> element.
    - Should not contain any content that is repeated across documents such as sidebars, navigation links, copyright information, site logos, and search forms.
    <footer> - Defines a footer for a document or section
    - Typically contains:
    • authorship information
    • copyright information
    • contact information
    • sitemap
    • back to top links
    • related documents
    - Can have several <footer> elements in one document
    <nav> - Defines a set of navigation links
    - Intended only for major blocks of navigation links
    <section> Defines a section in a document
    <table> - Consists of one <table> element, with one or more <tr>, <th>, <td>
    - The <tr> element defines a table row, the <th> element defines a table header, and the <td> element defines a table cell.
    - May also include <caption>, <colgroup>, <thead>, <tfoot>, and <tbody> elements.
    - See also
  • Basic Table
  • Tables - l
    • <thead> - Must have one or more <tr> tags inside
    - Used to group header content in an html table
    - Must be used as a child of a <table> element, after any <caption>, <colgroup> elements but before <tbody>, <tfoot>, and <tr> elements.
    The <thead>, <tbody>, and <tfoot> elements will not affect the layout of the table by default
    - See also
  • html Basics: Table
  • html Basics: Table - l
    • <tbody> - Must have one or more <tr> tags inside
    - Used to group the body content in an html table
    - Must be used as a child of a <table> element, after any <caption>, <colgroup>, <thead>.
    The <thead>, <tbody>, and <tfoot> elements will not affect the layout of the table by default
    - See also
  • html Basics: Table
  • html Basics: Table - l
    • <tfoot> - Must have one or more <tr> tags inside
    - Used to group footer content in an html table
    - Must be used in the following context: As a child of a <table> element, after any <caption>, <colgroup>, <thead>, and <tbody> elements.
    The <thead>, <tbody>, and <tfoot> elements will not affect the layout of the table by default
    - See also
  • html Basics: Table
  • html Basics: Table - l
    • <headings> - <h1> defines the most important heading
    - <h6> defines the least important heading
    - Only use one <h1> per page - this should represent the main heading/subject for the whole page
    - Do not skip heading levels - start with <h1>, then use <h2>, etc.
    <ol>: Ordered List - Can be numerical or alphabetical
    - <li> tag is used to define list items
    -Attributes: reversed - Specifies that the list order should be reversed (9,8,7...)
    - Attributes: start - Specifies the start value(number) of an ordered list
    - Attributes: type - Specifies the kind of marker to use in the list
    - See also:
  • Ordered Lists
  • Ordered Lists - l
    • <ul>: Unordered List - Defines an unordered (bulleted) list
    - Used with <li> to create a list
    - See also:
  • Unordered Lists
  • Unordered Lists - l
    • <li>: List Item - Defines a list item
    - Used inside ordered lists(<ol>), unordered lists (<ul>), and in menu lists (<menu>)
    - In <ul> and <menu>, the list items will usually be displayed with bullet points
    - In <ol>, the list items will usually be displayed with numbers or letters
    - Attributes: value - Only for <ol> lists. Specifies the start value of a list item. The following list items will increment from that number
    <pre> - Defines preformatted text
    - Text in a <pre> element is displayed in a fixed-width font, and the text preserves both spaces and line breaks.
    - The text will be displayed exactly as written in the html source
    - Attributes:
    • <code>: Defines a piece of computer code
    • <samp>: Defines a sample output froma computer program
    • <kbd>: Defines a keyboard input
    • <var>: Defines a variable
    <hr> - Defines a thematic break in an html page (e.g. a shift of topic)
    - Most often displayed as a horizontal rule that is used to separate content
    <noscript> - Defines an alternate content to be displayed to users that have disabled scripts in their browser or have a browser that doesn't support script.
    - Can be used in both <head> and <body>. When used inside <head>, the <noscript> element could only contain <link>, <style>, and <meta> elements
    <video> - Used to embed video content in a document, such as a movie clip or other video streams
    - Contains one or more <source> tags with different video sources. The browser will choose the first source it supports
    - The text between the <video> and </video> tags will only be displayed in browsers that do not support the <video> element
    - Supported video formats in html: MP4, WebM, and OGG
    - See also

    Inline Elements

    Element Value
    <a> - Defines a hyperlink, which is used to link from one page to another
    - If the <a> tag has no href attribute, it is only a placeholder for a hyperlink
    - See also: html Links html Links - l     		Default CSS: 
    b {
    font-weight: bold;
}
    <b> - Make text bold
    - Can also use CSS: "font-weight: bold;"
    <br> - Insert single-line texts
    - Is an empty tag, so no end tag     		Default CSS:
    <button> - A clickable button
    - See also: html Buttons html Buttons - l
    <cite> - Defines the title of a creative work (e.g. a book, a poem, a song, a movie, a painting, a sculpture, etc.)
    - The text in the <cite> element usually renders in italic     		-Default CSS: 
    cite {
    font-style: italic;
}
    <code> - Define a piece of computer code. The content inside is displayed in the browser's default monospace font
    <em> - Define emphasized text. The content inside is typically displayed in italic Default CSS: 
    em {
    font-style: italic;
}
    <i> - Defines a part of text in an alternate voice or mood. The content inside is typically displayed in italic
    - Often used to indicate a technical term, a phrase from another language, a thought, a ship name, etc
    - Use the <i> element only when there is not a more appropriate semantic element, such as:
    • <em> (emphasized text)
    • <strong> (important text)
    • <mark> (marked/highlighted text)
    • <cite> (the title of a work)
    • <dfn> (a definition term)
    		Default CSS: 
    i {
    font-style: italic;
}
    <img> - Used to embed an image in an html page
    - Has two required attributes:
    • src - Specifies the path to the image
    • alt - Specifies an alternate text for the image, if the image for some reason cannot be displayed

    - Always specify the width and height of an image. If width and height are not specified, the page might flicker while the image loads
    - To link an image to another document, simply nest the <img> tag inside an <a> tag
    - See also: html Images     		Default CSS: 
    img {
    display: inline-block;
}
    <input> - Specifies an input field where the user can enter data
    - The most important form element
    - Can be displayed in several ways, depending on the type attribute
    - See alsp html Forms
    <label> - Defines a label for several elements
    - The for attribute of <label> must be equal to the id attribute of the related element to bind them together. A label can also be bound to an element by placing the element inside the <label> element
    - Attributes:
    • for: element-id
      - Specifies the id of the form element the label should be bound to
    • form: form-id
      - Specifies which form the label belongs to
    		- Default CSS: 
    label {
    cursor: default;
}
    <output> - Used to represent the result of a calculation (like one performed by a script)
    - Attributes:
    • for: element-id
      - Specifies the relationship between the result of the calculation, and the elements used in the calculation
    • form: form-id
      - Specifies which form the output element belongs to
    • name: name
      - Specifies a name for the output element
    		Default CSS: 
    output {
    display: inline;
}
    <q> - Defines a short quotation
    - Normally insert quotation marks around the quotation
    - Use <blockquote> for long quotations
    Attributes:
    • cite: URL
      - Specifies the source URL of the quote
    		Default CSS: 
    q {
    display: inline;
}

    q:before {
    content: open-quote;
}

    q:after {
    content: close-quote;
}
    <script> - Used to embed a client-side script (JavaScript)
    - Either contains scripting statements, or it points to an external script file through the src attribute
    - Common uses for JavaScript are image manipulation, form validation, and dynamic changes of content
    - Also look at the <noscript> element for users that have disabled scripts in their browser, or have a browser that doesn't support client-side scripting
    - See also: html Scripts
