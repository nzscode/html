    Autofocus

    • Is a boolean attribute
    • When present, it specifies that a button should automatically get focus when the page loads
    Example
    HTML
    <button type="button" autofocus> 
    Autofocus Button 
</button>


    Disabled

    • Is a boolean attribute
    • When present, it specifies that the button should be disabled
    • Is unusable and un-clickable
    • Can be set to keep a user from clicking on the button until some other condition has been met (like selecting a checkbox, etc.). Then, a JavaScript could remove the disabled value, and make the button clickable again
    Example
    HTML
    <button type="button" disabled> 
    Disabled Button 
</button>


    Form

    • Specifies the form the button belongs to
    • Value of this attribute must be equal to the id attribute of a <form> element in the same document
    Example
    HTML
    <form action="" id="formID"> 
    <button type="button" form="formID"> 
        Form Button
    </button>
</form>


    Formaction

    • Specifies where to send the form-data when a form is submitted. This attribute overrides the form's action attribute.
    • Only used for buttons with type="submit"
    • Possible values:
      • An absolute URL - the full address of a page (like href="http://www.example.com/formresult.asp")
      • A relative URL - points to a file within the current site (like href="formresult.asp")
    Example
    HTML
    <form action="/action-page.php">
    <button
        type="submit"
        formaction="/action-page2.php"
    >
        Formaction Button
    </button>
</form>


    Formmethod

    • Specifies which HTTP method to use when sending the form-data. This attribute overrides the form's method attribute.
    • Only used for buttons with type="submit"
    • Can be sent as URL variables (with method="get") or as HTTP post (with method="post")
      • get: Appends the form-data to the URL: URL?name=value&name=value
      • post: Sends the form-data as an HTTP post transaction
    • Notes on the "get" method:
      it appends the form-data to the URL in name/value pairs
      it is useful for form submissions where a user want to bookmark the result
      There is a limit to how much data you can place in a URL (varies between browsers), therefore, you cannot be sure that all of the form-data will be correctly transferred
      Never use the "get" method to pass sensitive information! (password or other sensitive information will be visible in the browser's address bar)
    • Notes on the "post" method:
      it sends the form-data as an HTTP post transaction
      Form submissions with the "post" method cannot be bookmarked
      it is more robust and secure than "get"
      it does not have size limitations
    Example
    HTML
    <form action="/action-page.php" method="get">
    <button type="submit">
        Formmethod Button
    </button>
</form>


    Formnovalidate

    • Is a boolean attribute
    • When present, it specifies that the form-data should not be validated on submission
    • Overrides the form's novalidate attribute
    • Only used for buttons with type="submit"
    Example
    HTML
    <form action="/action-page.php" formnovalidate>
    <button type="button">
        Fromnovalidate Button
    </button>
</form>


    Formtarget

    • Specifies where to display the response after submitting the form
    • This attribute overrides the form's target attribute
    • Only used for buttons with type="submit"
      • -blank: Loads the response in a new window/tab
      • -self: Loads the response in the same frame (this is default)
      • -parent: Loads the response in the parent frame
      • -top: Loads the response in the full body of the window
      • framename: Loads the response in a named iframe
    Example
    HTML
    <form action="/action-page.php" formtarget="-blank";
    <button type="submit">
        Formtarget Button -blank
    </button>
</form>
    <form action="/action-page.php" formtarget="-self";
    <button type="submit">
        Formtarget Button -self
    </button>
</form>
    <form action="/action-page.php" formtarget="-parent";
    <button type="submit">
        Formtarget Button -parent
    </button>
</form>
    <form action="/action-page.php" formtarget="-top";
    <button type="submit">
        Formtarget Button -top
    </button>
</form>
    <form action="/action-page.php" formtarget="-framename";
    <button type="submit">
        Formtarget Button -framename
    </button>
</form>


    Popover

    Example
    HTML

    This is a popover 

    <h1 popover id="popoverID">This is a popover</h1>
<button popovertarget="popoverID">
    Popover Button
</button>


    Popovertargetaction

    • Allows you to define what happens when you click the button
    • Default is toggle
      • show: The popover element is showed when you click the button, and remains open after.
      • hide: The popover element is hidden(never shown) when you click the button
      • toggle: Default value. The popover element is toggled between hidding and showing when you click the button
    Example
    HTML

    This is a popovertargetaction 

    <h1 popover id="popovertargetactionIDshow">This is a popovertargetaction</h1>
<button popovertarget="popovertargetactionIDshow" popovertargetaction="show">
    Popovertargetaction Button: show
</button>

    This is a popovertargetaction 

    <h1 popover id="popovertargetactionIDhide">This is a popovertargetaction</h1>
<button popovertarget="popovertargetactionIDhide" popovertargetaction="hide">
    Popovertargetaction Button: hide
</button>

    This is a popovertargetaction 

    <h1 popover id="popovertargetactionIDtoggle">This is a popovertargetaction</h1>
<button popovertarget="popovertargetactionIDtoggle" popovertargetaction="toggle">
    Popovertargetaction Button: toggle
</button>


    Name

    • Specifies the name for a <button> element
    • Used to reference form-data after the form has been submitted, or to reference the element in a JavaScript
    • Several <button> elements can share the same name. This allows you to have several buttons with equal names, which can submit different values when used in a form
    Example
    HTML
    <form action="/action-page.php" method="get">
    <button name="buttonName" type="submit">
        Name Button
    </button>
</form>


    Type

    • Specifies the type of button
    • Always specify the type attribute for the <button> element
    • Different browsers may use different default types for the <button> element
      • button: The button is a clickable button
      • submit: The button is a submit button (submits form-data)
      • reset: The button is a reset button (resets the form-data to its initial values)
    Example
    HTML
    <form action="/action-page.php" method="get">
    <button type="button">
        Type Button
    </button>
</form>
    <form action="/action-page.php" method="get">
<label for="firstName">First Name:</label>
<input
    type="text"
    name="firstName"
    id="firstName"
/>
    <button type="submit">
        Type Button: submit
    </button>
</form>
    <form action="/action-page.php" method="get">
<label for="firstName">First Name:</label>
<input
    type="text"
    name="firstName"
    id="firstName"
/>
    <button type="reset">
        Type Button: reset
    </button>
</form>


    Value

    • Specifies the initial value for a <button> in an HTML form
    • In a form, the button and its value is only submitted if the button itself was used to submit the form
    Example
    HTML
    <form action="/action-page.php" method="get">
    <button type="submit" value="val-Button">
        Value Button
    </button>
</form>
