Basic Form

  • Used to collect user input
  • container for different types of input elements, such as: text fields, checkboxes, radio buttons, submit buttons, etc.
Example
HTML



<form action="">
    <label for="First_Name1">First Name: </label>
    <input type="text" id="First_Name1">
    <br>
    <label for="Last_Name1">Last Name: </label>
    <input type="text" name="Last_Name1" id="Last_Name1">
    <br>
    <input type="submit">
</form>

Default CSS

  • The Dault CSS for the <input> fields, is unique for each browser

form {
    display: block;
    margin-top: 0em;    
}

Default Label CSS

label {
    cursor: default;
}

Form Attributes

Type Value Description
Action action - Defines the action to be performed when the form is submitted
- "action_page.php" contains server-side script that handles the form data
- If the "action" attribute is left blank, the action is set to the current page
Target _blank
_self
_parent
_top
_iframe 		- Specifies where to display the response that is received when the form is submitted
Method method - Specifies the http method to use when submitting form data
- Default http method is "method = 'get'"
- Can also use http post "method = 'post'"

Notes on GET:
Appends the form data to the URL, in name/value pairs
NEVER use GET to send sensitive data! (the submitted form data is visible in the URL!)
The length of a URL is limited (2048 characters)
Useful for form submissions where a user wants to bookmark the result
GET is good for non-secure data, like query strings in Google

Notes on POST:
Appends the form data inside the body of the HTTP request (the submitted form data is not shown in the URL)
POST has no size limitations, and can be used to send large amounts of data.
Form submissions with POST cannot be bookmarked
Autocomplete autocomplete - Specifies if autocomplete should be on or off
- When on, browser completes the form on data user has already used
Novalidate novalidate - A boolean attribute
- Specifies that the form data (input) shouldn;t be validated on submission
Name name Specifies the name of the form
Rel rel Specifies the relationship between a linked resource and the current document

Target

  • Specifies where to display the response that is received after submitting the form.
  • Default value is _self
  • Possible Values:
    • _blank The response is displayed in a new window or tab
    • _self The response is displayed in the current window
    • _parent The response is displayed in the parent frame
    • _top The response is displayed in the full body of the window
    • framename The response is displayed in a named iframe
Type

</form>

</form>

- Users will see a drop-down list of the pre-defined options as they input data
- list attribute of the <input> element, must refer to the id attribute of the <datalist> element

Input

  • Default value is text
  • Possible Values:
    • text: single line text input field
    • button: defines a button
    • checkbox: lets user select none or more option of a limited number of options
    • color: is used for input fields to show a color
      - Depending on the browser a color picker may be available
    • date: pick a date from a calendar
    • datetime-local: from a calendar Pick a date & time
    • email: Field to enter an email address, will be automatically validated on submit
    • file: browse and select a file
    • hidden: - an invisible(to user) field
      - often stores what database record that needs to be updated for when the form is submitted
    • image: defines an image as a submit button
    • month: pick a month & a year (no time zone)
    • number: Field for entering a number
      - can set restrictions on acceptable numbers
      - can use attributes (min, max, step (the number of intervals), value (specifies a default value))
    • password: - field for password entry
      - characters are masked
    • radio: selection of radio buttons, only allows one selection
    • range: - control for entering a number where the exact value is not important
      - default range is 0 to 100, but restrictions can be set. br - attributes (min, max, step)
    • reset: resets all the values in the form
    • search: define a search field (site search, or search from a specific site)
    • submit: a submit button
    • tel: - field for enetering a telephone number
      - can use RegEx pattern for validation
    • time: pick a time (no time zone)
    • url: field for entering a URL
    • week: choose a week and date from a calendar (no time zone)


  • Restricitons:
    • checked: specifies that an input field should be pre-selected when the page loads (for type="checkbox" or type="radio")
    • disabled: specifies that an input field should be disabled
    • max: specifies the maximum value for an input field
    • maxlength: specifies the maximum number of character for an input field
    • min: specifies the minimum value for an input field
    • pattern: specifies a regular expression to check the input value against
    • readonly: specifies that an input field is read only (cannot be changed)
    • required: specifies that an input field is required (must be filled out)
    • size: specifies the width (in characters) of an input field
    • step: specifies the legal number intervals for an input field
    • value: specifies the default value for an input field

Text Area

  • Allows the entry of multiple text lines
  • "rows" attribute specifies the visible number of lines in the text area
  • "cols" specifies the visible width of the text area
  • "maxlength" specifies the maximum number of words allowed
  • Size of the text area can also be specified through CSS
Example
HTML
<form action="">
    <label for="complaint">
        Please write your complaint
        below:
    </label>
    <textarea
        name="complaint"
        id="complaint"
        cols="25"
        rows="5"
        placeholder="Enter text here:"
        max-length = "100"
    ></textarea>
</form>

Input Attributes

Type Value Description
Value value Read only input field, that can't be changed
Will be sent when submitting the form
Readonly readonly Read only input field, that can't be changed
Will be sent when submitting the form
Disabled disabled Field is unusable and unclickable
Size size Specifies the visible width in characters of the input field
Works with text, search, tel, url, email, password
Default size is 20
Maxlength maxlength Specifies the maximum number of characters allowed in an input field
When set, the input field won't accept more than the specified number of characters
To provide feedback on this, JS code must be written
Min & Max
min
max 		Minimum & Maximum values for any field
Works with number, range date, datetime-local, month, time, and week
Multiple multiple Specifies that more than one value can be entered
Works with email and file
Pattern pattern Specifies a RegEx that the input field value is checked against, when the form is submitted
Works with text, date, search, url, tel, email, and password
Placeholder placeholder A hint or example that shows what kind of input is expected
Is displayed in the input field
Works with text, seacrh, url, number, tel, email and password
Required required Specifies that an input field must be filled out before submitting the form
Works with text, search, url, tel, email, password, date pickers, number, checkbox, radio, and file.
Step step Specifies the legal number intervals for an input field
Can be used together with the "max" and "min" attributes to create a range of legal values
Works with number, range, date, datetime-local, month, time and week
Autofocus autofocus Specifies that an input field should automatically get focus when the page loads
Height & Width height
width 		Specify the height and width of an input type="image" element
Always specify both the height and width attributes for images
Datalist list Refers to a datalist element that contains pre-defined options for an input element
Autocomplete autocomplete Specifies whether a form or an input field should have autocomplete on or of
When a user starts to type in a field, the browser should display options to fill in the field, based on earlier typed values
Works with form and text, search, url, tel, email, password, datepickers, range, and color
