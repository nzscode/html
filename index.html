      • Form Attributes
          • Type Explanation Example HTML Syntax
          Action - Defines the action to be performed when the form is submitted
          - "action_page.php" contains server-side script that handles the form data
          - If the "action" attribute is left blank, the action is set to the current page 



          <form action="./action_page.php">
    <label for="f_name">
        First Name:
    </label>
    <br />
    <input
        type="text"
        name="f_name"
        id="f_name"
        value="Jane"
    />
    <br />
    <label for="l_name">
        Last Name:
    </label>
    <br />
    <input
        type="text"
        name="l_name"
        id="l_name"
        value="Doe"
    />
</form>
          Target - Specifies where to display the response that is received when the form is submitted
          Value Description
          _blank Response is displayed ina anew window or tab
          _self Response is displayed in current window
          _parent Respons is displayed in the parent frame
          _top Response is displayed in the full body of the window
          iframe Response is displayed in a named iframe 
          <form action="./action_page.php" target="_blank" >
    <label for="name">Enter whole Name:
    </label>
    <input
        type="text"
        name="name"
        id="name"
    />
    <input type="submit" />
</form>
          Method - Specifies the http method to use when submitting form data
          - Default http method is "method = 'get'"
          - Can also use http post "method = 'post'"
          Notes on GET:

          Appends the form data to the URL, in name/value pairs
          NEVER use GET to send sensitive data! (the submitted form data is visible in the URL!)
          The length of a URL is limited (2048 characters)
          Useful for form submissions where a user wants to bookmark the result
          GET is good for non-secure data, like query strings in Google

          Notes on POST:
          Appends the form data inside the body of the HTTP request (the submitted form data is not shown in the URL)
          POST has no size limitations, and can be used to send large amounts of data.
          Form submissions with POST cannot be bookmarked 



          <form action="./action_page.php" method="get"></form>

<form action="./action_page.php" method="post"></form>
          Autocomplete - Specifies if autocomplete should be on or off
          - When on, browser completes the form on data user has already used 
          <form action="/action_page.php" autocomplete="on"></form>
          No Validate - A boolean attribute
          - Specifies that the form data (input) shouldn;t be validated on submission 
          <form action="/action_page.php" novalidate></form>
