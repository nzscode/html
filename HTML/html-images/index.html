Basic Image

  • The tag is empty.
  • Contains attributes only, and does not have a closing tag.
  • Has two required attributes:
    • src - Specifies the path to the image
    • alt - Specifies an alternate text for the image
  • Can also have a size attribute:
    • Width & Height can be set on the image using the style attribute (Recommended)
    • Width & Height can also be set through the Stylesheet. If setting via Stylesheet, remeber stylesheets are not hardcoded, whereas the style attribute is. To use Stylesheet, follow the CSS given
Examples
HTML
<img 
    src="../../images/img.png" 
    alt="Star Image"
    style="width: 25px; 
            height: 25px" />
Default CSS
img {
    width: 25px; 
    height: 25px
}




Image Attributes

alt

  • The text should describe the image if the image contains information
  • The text should explain where the link goes if the image is inside an <a> element
  • Use alt="" if the image is only for decoration
Examples
HTML
<img src="../../images/img.png" alt="star-image" />


Height

  • Always specify both the height and width attributes for images. If height and width are set, the space required for the image is reserved when the page is loaded. However, without these attributes, the browser does not know the size of the image, and cannot reserve the appropriate space to it. The effect will be that the page layout will change during loading (while the images load).
  • Downsizing a large image with the height and width attributes forces a user to download the large image (even if it looks small on the page). To avoid this, rescale the image with a program before using it on a page.
Examples
HTML
<img
    src="../../images/img.png"
    alt="Star-Image"
    height="100"
/>


Loading

  • eager: Default. Loads an image immediately
  • lazy: Defer loading of images until some conditions are met
Examples
HTML
<img
    src="../../images/img.png"
    alt="Star-Image"
    height="100"
    loading="lazy"
/>


Longdesc

  • A hyperlink to a detailed description of an image.
  • Possible values:
    • An id to another element
    • An absolute URL - points to another web site (like longdesc="http://www.example.com/description.txt")
    • A relative URL - points to a file within a web site (like longdesc="description.txt")
Examples
HTML
<img
    src="../../images/img.png"
    alt="Star-Image"
    height="100"
    longdesc="https://www.example.com/explanation"
/>


Src

  • Absolute URL - Links to an external image that is hosted on another website. Example: src="https://www.w3schools.com/images/img-girl.jpg".
  • Relative URL - Links to an image that is hosted within the website. Here, the URL does not include the domain name. If the URL begins without a slash, it will be relative to the current page. Example: src="img-girl.jpg". If the URL begins with a slash, it will be relative to the domain. Example: src="/images/img-girl.jpg".
  • It is almost always best to use relative URLs. They will not break if you change domain
  • A broken link icon and the alt text are shown if the browser cannot find the image
Examples
HTML
<img 
    src="../../images/img.png" 
    alt="Star-Image"
    height="100" 
/>


Width

  • Always specify both the height and width attributes for images. If height and width are set, the space required for the image is reserved when the page is loaded. However, without these attributes, the browser does not know the size of the image, and cannot reserve the appropriate space to it. The effect will be that the page layout will change during loading (while the images load)
  • Downsizing a large image with the height and width attributes forces a user to download the large image (even if it looks small on the page). To avoid this, rescale the image with a program before using it on a page
Examples
HTML
<img
    src="../../images/img.png"
    alt="Star-Image"
    height="100"
    width="100"
/>




Figure & FigCaption

  • Figure
    • Specifies self-contained content, like illustrations, diagrams, photos, code listings, etc.
    • While the content of the <figure> element is related to the main flow, its position is independent of the main flow, and if removed it should not affect the flow of the document
    • The <figcaption> element is used to add a caption for the <figure> element
  • FigCaption
    • Defines a caption for a <figure> element
    • Can be placed as the first or last child of the <figure> element
Examples
HTML
<figure>
<img 
    src="../../images/img.png" 
    alt="star image" 
    style="width:100%">
    <figcaption>
        Fig. 1 Pic of a star with the words Image
    </figcaption>
</figure>
CSS
figcaption {
    display: block;
}
